Twitter-Clone

Twitter-Clone is an open-source simplified version of the Twitter/X social media platform developed by mateuszwszola. It is written with React (front), Node (back) and MongoDB (database).

JMeter test code

  • Objectives

    The aim of this project is to explore the scalability of Twitter-Clone. More specifically, I will answer the question: Does Twitter-Clone support a load of 1000 users accessing and actively using the platform at the same time?

  • Scenario outline

    To answer this question, I adopted the following scenario plan:

    • 1. With a pool of 1000 distinct users
    • 2. Each user will perform the following 3 actions in sequence:
      •     a. Log-in to the Twitter-Clone platform
      •     b. Create and post a Tweet
      •     c. Visit the Home Page to read tweets
    • 3. I will include a ramp-up period of 20s (i.e. every 20ms, a new user will login)

  • Evaluation criteria and Expected result

    I will consider that Twitter-Clone is scalable based on the following benchmark for each request:

    •     1. The user receives a successful response (there's no error coming from the server)
    •     2. The response time is less than 500ms

  • Scenario implementation

    I implemented the scenario using JMeter. You can access the test plan file in my Github repository.

  • Performance Test Results

    Twitter-Clone failed to scale with 1000 users. Actually, starting with 300 users, the response time of each request exceeded the required 500ms, as shown in the graph below.

    Although, I note as positive points:
        1. Even under high load, the application always returned the correct response.
        2. Following the load peak, the application returned to an acceptable response time, as shown in the graph below.

  • Recommendations

    1. For more accurate results, performance tests should be rerun on a more powerful server (the tests above were done on my Laptop).
    2. Developers should work on identifying and correcting the performance issues and bottlenecks.
