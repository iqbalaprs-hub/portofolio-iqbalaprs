Twitter-Clone

Twitter-Clone is a social media platform where the user can share his thoughts and opinion by tweeting. Users can see others’ tweets and discuss with them diverse topics, news and opinions through likes and replies.

Specifications

To begin with, I familiarized myself with Twitter-Clone by exploring its funtionalities. I then wrote down a specification of its features, in order to provide me with a guide for writing test cases. Here are the Twitter-Clone specifications:

Twitter-Clone specifications (Click to reveal)

  • 1- Sign Up:
  • 1- Only anonymous user can see the “Sign Up” button in the navigation bar
      • 2- When the user clicks on “Sign Up” button, the “Sign Up” form appears:
    • - The “Sign Up” form contains the following inputs:
      • a. Name:
        • i. String
        • ii. Multiple users can have the same name
      • b. Username:
        • i. String
          • - Only Period “.”, Dash “- “and Underscore “_” are allowed as special characters
        • ii. Username is unique. Two users cannot have the same username
        c. Email:
        • i. String in the form name@example.com
        • ii. Email is unique. Two users cannot have the same email
        d. Password
        • i. Must be at least 8 characters with at least 1 letter and 1 number
        e. Confirm password
        • i. Must be exactly equal to the password
      - The link “Sign in now” send you to the Sign in page
      3- When the user submits the “Sign Up” form, by clicking on the “Sign Up” button:
    • - The user’s account is created
    • - The user is automatically signed in
    • - The user is taken to the “Home” page
  • 2- Sign In:
      1- When the user clicks on “Sign In” button, the “Sign In” form appears:
    • - The “Sign In” form contains the following inputs:
      • a. Email or username
      • b. Password
    • - The link “Sign up now” send you to the Sign-Up page
      2- When the user submits the “Sign In” form, by clicking on the “Sign In” button:
    • - If the username or email, and password are correct:
      • a. The user enters his account (the user is signed-in)
      • b. The user is taken to the “Home” page
      • c. The “Sign In” button in the navigation bar disappear (Only anonymous user can see the “Sign In” button in the navigation bar)
    • - else an error message “Invalid login credentials” will appear if username, email or password is written wrong
      • a. An error message “Username is required” if username or email is empty
      • b. An error message “Password is required” if password is empty
  • 3- Current user's personal menu
  • 1- This menu is no visible if the user is anonymous. It is only visible for signed in users
      • 2- When the user clicks on “Sign Up” button, the “Sign Up” form appears:
    • - Name and username (Both assert the identity of the account)
    • - Profile
      • a. The user can access his profile
    • - Settings
      • a. The user can edit his profile
      • b. The user can delete his account
        • i. The user can cancel the deletion of his account
    • - Sign Out
      • a. The user can sign out
  • 4- Tweet
  • 1- The tweet button is not visible if the user is anonymous. It is only visible for signed in users
      • 2- When the user clicks on the tweet button, a modal appears:
      • - A text box
        • i. The text box has the placeholder “What’s happening?”
      • - Tweet submit button
        • i. Disabled
          • - if the text box is completely empty
          • - If the text box has more than 280 characters
        • ii. Active, if the text box has at least one character (space included)
      • - The “X” button
        • i. The user can exit the tweet by clicking on “X”
  • 3- When the user writes on the text box, he clicked the tweet submit button, the tweet is created
  • 4- Tweets appear in the “Home” page and in the user’s profile
      • 5- A tweet is displayed with:
    • - Name and username of the author
    • - The tweet date publication
    • - The text
        • - Reply button
      • i. Anonymous user cannot reply
          • ii. The user can reply to any tweet more than once
        • - Each time a user reply to the tweet, the number next to the bubble increases by one
        • - The number shows how many replies for a specific tweet
        • - The replies are sorted from newest (top) to oldest (bottom)
          iii. The user can delete only his reply
        • - Each time a user deletes the tweet, the number next to the bubble decreases by one
        • - The number shows how many replies for a specific tweet
        - Like option
      • i. Anonymous user cannot like
          • ii. The user cannot like a tweet more than once
          • - Each user liking the tweet, the number next to the like increases by one
          • - This number shows the number of users liking the tweet
          iii. The user can remove the like
          • - Each user removing the like, the number next to the like decreases by one
          • - This number shows the number of users liking the tweet
  • 6- The user can delete his own tweet. The user cannot delete other users’ tweets
  • 5- Home
  • 1- Anonymous and signed in users can see “Home” page
      • 2- The user clicks on “Home” button in the navigation bar:
      • i. The tweets are organized from newest (top) to oldest (bottom)
      • ii. When the user opens the “Home” page, The last 10 tweets are displayed
      • iii. When the user scrolls down, the next 10 tweets appear
  • 6- All profiles
  • 1- Anonymous and signed in users can see “All profiles”
      • 2- The user clicks on the “All profiles” button in the navigation bar :
      • i. You can see all twitter’s users (The number of users present is equal to the number of users having an account)
      • ii. You can recognize each user by the name, username and avatar picture
      • iii. You can access the profile of each user
      • IV. The names are sorted from oldest member to newest member (left to right; top to bottom)
  • 7- Profile
  • 1- Anonymous user and signed in users can see any user’s profile
  • 2- Only a user can modify his own profile
      • 3- You can see the personal information of the user:
    • - Name
    • - Username
    • - The day the user joined Twitter
        • - Follow option (the feature “follow” is not completely implemented):
        • i. Follow
        • ii. Unfollow
        - Following
        • i. The number shows how many accounts is the user following
        - Followers
        • i. The number shows how many accounts is following the user
    • - All the user’s tweets
    • - All the user’s tweet liked
    • - Avatar picture
    • - Background image
    • - “Edit profile” that enable you to change the personal information
  • 8- Edit profile
  • 1- When the user enters the “Edit profile” page, he can fill the “Edit profile” form which contains:
      - Name
      • i. String
      • ii. Multiple users can have the same name
      • iii. The new name replaces the old name
      • IV. It is not allowed to be empty
      - Bio
      • i. String
      - Location
      • i. String
      - Website
      • i. It has to be a valid URL
      - Avatar (URL)
      • i. It has to be a valid URL
      - Background image (URL)
      • i. It has to be a valid URL
      • 2- If the user has no more things to change in the form, he can either:
      • - Update his profile by clicking on the “Update profile” button
      • - Go back to the profile without updating the changes by clicking on the “Go back” button
