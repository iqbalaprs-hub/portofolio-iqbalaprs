Billzer

Billzer is a web application that helps users manage shared expenses. It offers features to input expenses paid by group members, and the platform automatically calculates the amount owned by each individual.
The objective of this project is to write comprehensive end-to-end tests to evaluate the quality of this app and report bugs if found.

Cypress tests - Test run

  • Specifications

    To begin with, I familiarized myself with Billzer by exploring its funtionalities. I then wrote down a specification of its features, in order to provide me with a guide for writing test cases. Here are the Billzer specifications:

    Specifications+

    • 1. The user can create a calculation by filling:
      • a. Member count:
        • 1. The Member count is an integer
        • 2. The Member should be between 2 and 40
      • b. Name of the Calculation:
    • 2. Inside a calculation:
      • a. The Calculation name is displayed on Top
      • b. for each Member, there is a Form
    • 3. In each form:
      • a. The user can:
        • 1. Fill the Person's name
        • 2. Fill the Expense's name
        • 3. Fill the Expense's amount:
          • • Positive number (Integer or decimal)
        • 4. Can add or remove expense:
          • • You cannot delete all expenses.
          • • Only 1 input field of expense for each person must remain.
          • • If the person has no expense, the expense’s amount must be zero
        • 5. Can include or exclude persons from an expense:
          • • In any Expense, at least one member must be included
          • • If any member of the expense is excluded, the 2 Person Icon is colored in orange
          • • If no member of the expense is excluded, the 2 Person Icon remains grey
      • b. The total amount of the list of expenses paid is displayed next to the person name
    • 4. When changing any value, an orange bar appears saying: “Unsaved changes. Hit the calculate button below or click here"
    • 5. In order to calculate, you either:
      • a. Click the blue button “Calculate and Save”
      • b. Click the orange bar saying: “Unsaved changes. Hit the calculate button below or click here”
    • 6. When clicking on Calculate, the app calculates how much each Member owes to the other
      • a. The Calculation is done so that, when every member pays what he owes to the other, all members have paid the same amount
        • 1. Formula or Process or Algorithm to Calculate what each member owes to the other (given by the product owner)
      • b. There are 2 ways the user can get the link of the calculation:
        • 1. The orange bar is replaced by a green sentence saying that the calculation is saved and it has provided you a link of the calculation
        • 2. A box under the calculation result appears. Inside it, a link is generated and can be shared, so that other users can check the Calculation
          • • A button allows the user to copy the link
          • • A link to Facebook and WhatsApp allows the user to share the link
      • c. Every time an expense is changed:
        • 1. The result of the Calculation is erased
        • 2. Orange bar reappears telling the user that there are unsaved changes
        • 3. The button “Calculate and Save!” reappears
    • 7. Inside the Calculation, a “New” button situated at the top left allows the user to create a new Calculation
      • a. When clicking on it, the user is redirected to the home page

  • Bug reports

    My tests found 5 bugs in the Billzer app. These bugs can be transferred to the development team for fixing.

    • Bug 1: Spaces are ignored in the calculation name+

      Summary

      When creating a calculation, spaces are sometimes ignored from the Calculation name.

      Steps to reproduce

      (1) Prereq. BILLZER page is already opened
      (2) Type in the Name input: 12345           !@#$%^&*()_+-{}|:?><
      (3) Click the button GO

      Actual result 🐞

      Spaces are ignored. The calculation is created with name: 12345!@#$%^&*()_+-{}|:?><

      Expected result

      At the top of the page, I see the name: 12345           !@#$%^&*()_+-{}|:?><

    • Bug 2: In manual tests, not in Cypress, typing a letter in the Member count input triggers a bug+

      Summary

      Member count input only accepts numbers. But the user can type the letter "e"

      Steps to reproduce

      (1) Prereq. BILLZER is already opened
      (2) Type in the Member count input: qwertyuioplkjhgfdsazxcvbnm
      (3) Click the button GO

      Actual result 🐞

      Member count input contains "e".

      Expected result

      Member count input is empty. A tooltip appears saying "Please enter a number."

    • Bug 3: When the Member count input is empty, an error tooltip should appear instead of displaying the error on a new page.+

      Summary

      The way of telling that the user cannot leave Member count input is wrong

      Steps to reproduce

      (1) Prereq. BILLZER page is already opened
      (2) Leave the Name input empty
      (3) Click the button GO

      Actual result 🐞

      A new page is created only for this notification "Only groups between 2 and 40 people are allowed".

      Expected result

      A tooltip appears saying "Please enter a number.".
      A page does not need to be created only for this notification.
      A tooltip is sufficient

    • Bug 4: You should not exclude all members from an expense+

      Summary

      You should not exclude all members from an expense, but the app is allowing it.

      Steps to reproduce

      Prereq. BILLZER is already opened
      In the calculation page,the user:
      - Type in Person's name in form 1: Person1
      - Type in Person's name in form 2: Person2
      - Type in Person's name in form 3: Person3

      Click on the 2-persons icon of the expense "Ticket" of the form Person1 and exclude Person1

      User was able to exclude Person1. All persons related to an expense are removed.

      Expected result

      The user is not allowed to exclude Person1. At least one person should be included in each expense
