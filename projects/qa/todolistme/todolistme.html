TodoListMe

TodoListMe is a web application that helps users organize their goals and tasks. It allows users to categorize their tasks, complete them or schedule them for a future date.

Specifications

To begin with, I familiarized myself with Todolistme by exploring its funtionalities. I then wrote down a specification of its features, in order to provide me with a guide for writing test cases. Here are the Todolistme specifications:

Todolistme specifications (Click to reveal)

  • 1. A user can manage lists:
  • 1. At first connection, 5 lists are present by default: “Today’s tasks”, “Projects”, “Things to do later”, “Example template”, “Important info”
    • 2. The user can create new list:
      • a. By clicking the icon “add new list”. When creating it, you can either:
        • i. Save it
        • ii. Cancel it
      • b. Copying an existing list. When copying it, you can either:
        • i. Save it
        • ii. Cancel it
        • iii. Name it:
          • 1. If the name is not given by the user, the copied list takes the original name and add “Copy of “at the beginning
          • 2. Multiple copied lists can have the same name
  • 3. The user can delete the list.
  • 4. The user can name the list when creating it. When naming it, you can either:
    • a. Save it
    • b. Cancel it
    • c. Name it yourself:
      • a. You can name it in Arabic and Chinese
      • b. You can name it using special characters
      • c. You cannot leave it without a name
      • d. You cannot use spaces as a name
  • 5. The user can rename the list. When renaming it, you can either:
    • a. Save it
    • b. Cancel it
    • c. Multiple lists can have the same name
  • 6. The user can reorder the lists by dragging
  • 7. Each list has:
    • a. Name
    • b. (Number of To-Do-list items / Number of Done-list items)
  • 2. A user can manage categories:
  • 1. Categories can be created. When creating it, you either:
    • a. Save it
    • b. Cancel it
    • c. Name it:
      • a. If the name is not given by the user, the default name will be “New Category”
      • b. Name it yourself:
        • i. You can also use Arabic and Chinese language
        • ii. You can use special characters
        • iii. You cannot leave the list without a name
        • iv. You cannot use spaces as a name
      • c. Multiple categories can have the same name
  • 2. Categories can be renamed. When renaming it, you either:
    • a. Save it
    • b. Cancel it
  • 3. Categories can be deleted
  • 4. Categories can be dragged
  • 5. The user can manage lists in the categories:
    • a. Insert lists into the categories
    • b. Change the order of the lists
    • c. Copy the lists
    • d. Delete the lists
  • 6. Categories can hide its lists
  • 3. Each list’s content is composed of :
  • a. To-do items:
    • a. Items that must be done today
    • b. If it is empty, there is a sentence “No items. Why not add one below.”
    • c. All items must be unchecked
  • b. Done items:
    • a. Items that are already done
    • b. If it is empty, there is a sentence “No done items”
    • c. All items must be checked
  • c. Scheduled items:
    • a. Is composed of 2 categories:
      • i. Tomorrow
        • 1. Items that must be done tomorrow
      • ii. Later
        • 1. Items that must be done any day after tomorrow
    • b. If both tomorrow and later are empty, there is a sentence “Drag todos onto the Tomorrow or later titles above to put them off till later.”
    • c. All items must be unchecked
  • 4. What can we do in each of the 3 parts mentioned just above?
      1. To-do items:
        a. The user can add new item:
        • i. By clicking and typing any name in the input with placeholder “Type and hit Enter to add”.
        • ii. By dragging an item from done-items or the schedule-items
        • iii. By unchecking an item in the done-items
        b. The user can name the item:
        • i. You can also use Arabic and Chinese language
        • ii. You can use special characters
        • iii. You cannot leave the list without a name
        • iv. You cannot use spaces as a name
        • v. You can have multiple items with same name
        c. The user can delete an item
      • i. By clicking on the red X
        d. The user can rename an item. When renaming it, you can either:
        • i. Save it
        • ii. Cancel it
        e. The user can sort items
        • i. The way he desires (normal) by dragging an item up and down to re-order the list
        • ii. Alphabetical order
        • iii. Randomly
        • iv. Selecting only the top 3
      2. Done items:
        a. The user can add new item:
        • i. Checking an item in the to-do-list
        • ii. Dragging an item from the to-do-items or the schedule-items
        b. The user can name the item:
        • i. You can also use Arabic and Chinese language
        • ii. You can use special characters
        • iii. You cannot leave the list without a name
        • iv. You cannot use spaces as a name
        • v. You can have multiple items with same name
        c. The user can delete:
        • i. An item by clicking on the red X
        • ii. All items by clicking on the trash icon
        d. The user can rename an item. When renaming it, you can either:
        • i. Save it
        • ii. Cancel it
      3. Scheduled items:
        a. The user can add new item:
        • i. Dragging an item from the to-do-items or done-items
        b. The user can name the item:
        • i. You can also use Arabic and Chinese language
        • ii. You can use special characters
        • iii. You cannot leave the list without a name
        • iv. You cannot use spaces as a name
        • v. You can have multiple items with same name
        c. The user can delete an item:
        • i. by clicking on the red X
        d. The user can rename an item. When renaming it, you can either:
        • i. Save it
        • ii. Cancel it
        e. The user can hide or expose the items by clicking on the blue arrow
  • 5. In what way can you schedule your items:
      1. Today (to-do items)
      • a. It is for goals you want to accomplish today. There is no additional information about it
      2. tomorrow (Scheduled items)
      • a. If you add an item in the “tomorrow” category, the item is added with tomorrow’s date above it (for example: if today is 28/09/2023; then the sentence above the item will be “Friday 29th September 2023)
      • b. If you add other items, all items will be under ONE date (it is written one time), which is tomorrow’s date
      • c. If it is empty, nothing is written next to the title “Tomorrow”
      • d. If it is NOT empty, a number between parentheses appear next the title “Tomorrow” showing how many items are in the tomorrow’s date
      3. Later (Scheduled items)
      • a. You can specify the date only if:
        • i. Dates today and before it cannot be selected
        • ii. Dates tomorrow and after can be selected
      • b. Dates are automatically sorted (from newest (top) to latest (bottom))
      • c. Each date is written only one time. Even if many items have same day, they are all under one same day
      • d. If it is empty, nothing is written next to the title “Later”
      • e. If it is NOT empty, a number between parentheses appear next the title “Later” showing how many items are in the later’s date
  • 6. Progress line:
      1. To-do-list:
        a. There are two lines below the title “to do:”
      • i. A line originally takes the full width. Isn’t affected by anything
      • ii. A line which is considered a progress line. The width of the line is proportional to the number of tasks done (The line starts from the right)
        • 1. If there are no items in the done-items, the line is invisible
        • 2. If for example, there are a total of 10 items in the list (to-do-items plus the done-items) and we have only 1 item in the Done-items, this line will be 1/10 the width of the line just above it.
        • 3. If all items are in the Done-items, the width of the line will be equal to the width of the line above it.
      2. Done-list:
        a. There are two lines below the title “Done:”
        • i. A line originally takes the full width. Isn’t affected by anything
        • ii. A line which is considered a progress line. The width of the line is proportional to the number of tasks done (The line starts from the left)
          • 1. If there are no items in the done-items, the line is invisible
          • 2. If for example, there are a total of 10 items in the list (to-do-items plus the done-items) and we have only 1 item in the Done-items, this line will be 1/10 the width of the line just above it.
          • 3. If all items are in the Done-items, the width of the line will be equal to the width of the line above it.
  • 7. Save:
    • 1. Even if get out from the page, and then enter it later, all information remains the same. Nothing is lost
    • 2. The list is being saved automatically and it is shown by the Save icon (It is a green colored square icon middle at the top).
    • 3. The next day, the items added for after-tomorrow becomes automatically an item for tomorrow

🐞 Bug reports

My tests found 6 bugs in the Todolistme app. These bugs can be transferred to the development team for fixing

  • Bug 1: When creating a list, the "cancel" button is acting like the "save" button.

    Summary

    When creating a list, the "cancel" button is acting like the "save" button.

    Steps to reproduce

    (1) Prereq: Todolistme page is already opened.
    (2) Prereq: Local storage is erased and the web application is returned to its default state.
    (3) Create new list by clicking the icon "add new list".
    (4) Click "cancel" button.

    Actual result 🐞

    The creation of the new list was not cancelled and it is also named "New List" (Which is the default name).

    Expected result

    No new list was created.

    Notes

    The same bug occurs when copying a list

  • Bug 2: When creating a category, the "cancel" button is acting like the "save" button.

    Summary

    When creating a category, the "cancel" button is acting like the "save" button.

    Steps to reproduce

    (1) Prereq: Todolistme page is already opened.
    (2) Prereq: Local storage is erased and the web application is returned to its default state.
    (3) Click on the icon "add new category" in order to create a new category.
    (4) Click "cancel" button.

    Actual result 🐞

    The creation of the new category was not cancelled and it is named "New Category" (which is the default name).

    Expected result

    No new category was created.

    Notes

    The same bug occurs when copying a category

  • Improvement: the ID should not change when dragging an item.

    Summary

    To simplify automated tests on Cypress, the ID should not change when dragging an item.

    Steps to reproduce

    (1) Prereq: Todolistme page is already opened.
    (2) Prereq: Local storage is erased and the web application is returned to its default state.
    (3) Create several items in todo-items.
    (4) Drag the items (preferably randomly) to the scheduled-items

    Actual result 🐞

    When an item is dragged to another section (for example from to-do-items to scheduled-items), its ID changes

    Expected result

    The item keeps its original ID regardless if it is in to-do-items, done-items or scheduled-items section.
